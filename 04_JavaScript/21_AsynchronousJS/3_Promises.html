Promises in Asynchronous JavaScript

What is a Promise?

A Promise is an object representing the eventual completion or failure of an asynchronous operation. It allows you to handle asynchronous operations more elegantly than callbacks, avoiding callback hell.

Promises have three states: pending, fulfilled, and rejected. Once a promise is fulfilled or rejected, it cannot change its state.

Promise States:
  • Pending: Initial state, neither fulfilled nor rejected.
  • Fulfilled: Operation completed successfully.
  • Rejected: Operation failed.

Basic Promise Example


              let myPromise = new Promise((resolve, reject) => {
                  let success = true; // Simulate success or failure
                  if (success) {
                      resolve("Promise resolved successfully!");
                  } else {
                      reject("Promise rejected.");
                  }
              });
              //Promise handler
              myPromise.then((message) => {
                  console.log(message);
              }).catch((error) => {
                  console.error(error);
              });

Promise Chaining Example

              
                
             new Promise((resolve, reject) => {
               resolve(2);  // starting value
             })
             .then(num => {
              console.log(num); // 2
               return num * 2;   // pass 4 to next
             })
             .then(num => {
               console.log(num); // 4
               return num * 2;   // pass 8 to next
             })
             .then(num => {
               console.log(num); // 8
             });
